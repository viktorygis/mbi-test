# MBI — Контент экранов приложения

> **Назначение:**
> Описание всех экранов и секций веб-приложения MBI-теста: структура, тексты, элементы UI, логика переходов.
> Документ предназначен для разработчиков, дизайнеров и контент-менеджеров.

---

## Маршруты и страницы

| Маршрут | Компонент | Назначение |
| ------- | --------- | ---------- |
| `/mbi-test/` | `MbiEntryPage` | Прохождение теста (интро → форма → вопросы → завершение) |
| `/mbi-result/:id` | `MbiResultPage` | Страница результатов по ID |

---

## Поток шагов `MbiEntryPage`

```
intro → form → questions → loading → resultLink
```

| Шаг | Экран | Триггер перехода |
| --- | ----- | ---------------- |
| `intro` | `IntroScreen` | Кнопка «Пройти тест» |
| `form` | `FormScreen` | Успешная отправка формы |
| `questions` | `QuestionsScreen` | Последний ответ + «Завершить» |
| `loading` | Заглушка | Автоматически, пока сохраняются данные |
| `resultLink` | Блок со ссылкой | После сохранения результата |

## Экран 3 — Вопросы (`QuestionsScreen`)

| Элемент | Контент |
| ------- | ------- |
| Информация | ФИО и Дата в шапке |
| Прогресс | Текст + процент + прогресс-бар |
| Текст прогресса (предв.) | `Предварительный вопрос N из M` |
| Текст прогресса (MBI) | `Вопрос N из 22 (всего M)` |
| Вопрос | Текст из `questions.json` |
| Варианты ответа | Radio-кнопки: Никогда / Очень редко / Иногда / Часто / Очень часто / Каждый день |
| Навигация | ← Назад · Вперёд → / Завершить ✓ |

**Клавиатурное управление:**

| Клавиша | Действие |
| ------- | -------- |
| `←` | Предыдущий вопрос |
| `→` / `Enter` | Следующий вопрос / Завершить |
| `1`–`6` | Выбор варианта ответа |

**Структура вопросов:**
- Сначала идут **предварительные вопросы** (возраст, занятость, приоритет) из `preliminaryQuestions`
- Затем **22 вопроса MBI** из `questions`
- Данные загружаются из `questions.json`

> ⚠️ Кнопка «Заполнить тестовые ответы» — отладочная. В `MbiEntryPage` передаётся `showTestFillButton={true}`.

### 5.2 `MbiScalesSection` — Шкалы и результаты

**Блок «Итоговый результат»:**

| Элемент | Контент |
| ------- | ------- |
| Заголовок `h2` | **Итоговый результат** |
| Подзаголовок | Чем выше балл, тем выше ваш уровень профессионального выгорания. |
| Иконка + название | `burnout-index.svg` + название из `burnoutConfig.title` |
| Уровень | Текстовый уровень, цвет из `SEG_COLORS` |
| Баллы | N / 132 |
| Линейка | Сегментированная (если есть `norms`) или простая `0%–100%` |
| Рекомендация | `rec.short` + список `rec.details` |

**Блок «Профиль выгорания»:**

| Состояние | Контент |
| --------- | ------- |
| Есть сочетания | Список тревожных сочетаний из `combinedInterpretation()` |
| Нет сочетаний | Тревожные сочетания по шкалам не выражены. |

**Блок «Расшифровка результата»** (три шкалы, отсортированы по приоритету проблемности):

| Элемент | Контент |
| ------- | ------- |
| Заголовок `h2` | **Расшифровка результата** |
| Иконка + название + уровень | Три колонки для каждой шкалы |
| Баллы | N баллов из M |
| Линейка | Сегментированная или простая |
| Подсказка | Только для `reduction`: «Чем выше балл — тем сильнее выражено снижение ощущения эффективности.» |
| Рекомендации | `rec.short` + `rec.details` |

**Блок «Когда стоит обратиться за поддержкой»:**

> Если усталость, раздражительность, проблемы со сном или ощущение беспомощности не проходят несколько недель и начинают мешать жизни — имеет смысл обсудить это со специалистом. Это обычный и рабочий способ поддержки.

### 5.3 `MbiInformationSection` — Справочная информация

| Элемент | Контент |
| ------- | ------- |
| Заголовок `h2` | **Справочная информация** |

**Четыре карточки:**

**Эмоциональное истощение:**
- Что измеряет: Насколько человек эмоционально, физически и психически истощён.
- Как проявляется: Постоянная усталость / Нехватка сил / Раздражительность
- Как понимать: Высокие — признак выгорания / Средние — риск перегрузки / Низкие — ресурс сохранён
- Практический смысл: Если высокая — нужен отдых и снижение нагрузки.

**Деперсонализация:**
- Что измеряет: Степень эмоциональной дистанции в общении.
- Как проявляется: Холодное отношение / Снижение эмпатии / Общение без вовлечённости
- Как понимать: Высокие — дистанцирование / Средние — признаки отчуждения / Низкие — эмпатия сохранена
- Практический смысл: Рост шкалы — защита через дистанцию от перегрузки.

**Редукция личных достижений** ⚠️ обратная шкала:
- Что измеряет: Насколько человек чувствует себя компетентным и значимым.
- Как проявляется: «Я плохо справляюсь» / Обесценивание результатов / Снижение мотивации
- Как понимать: Низкие — более выраженная редукция / Высокие — эффективность сохранена
- Практический смысл: Чем ниже балл — тем сильнее выгорание.

**Как использовать шкалы вместе:**
- Высокое истощение + высокая деперсонализация → активная фаза выгорания
- Высокое истощение + снижение достижений → риск утраты мотивации
- Высокие по всем трём → выраженный синдром выгорания

Кратко: **Истощение** — «нет сил» · **Деперсонализация** — «отстраняюсь» · **Редукция** — «я не справляюсь»
